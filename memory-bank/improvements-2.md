## 功能调整


### 总览页面前端调整


**1. 调整卡片层级（将关键数据前置）**
*   **现状：** 首屏占据最大面积的是“看板”和“公告”，但目前它们都是**空状态**。用户登录后真正关心的核心数据（待办事项“3”，任务跟踪“0”）被挤到了屏幕中间甚至下方。
*   **建议：** 将中间那一排**数据统计卡片（待办3、任务0）提到页面的最顶端**。对于内部协同工具，用户第一眼需要知道的是“我今天要做什么”。
*   **重构后的顺序推荐：**
    1.  **核心指标层：** 待办数量、进行中任务、未读消息等（横向小卡片）。
    2.  **快捷操作层 / 动态层：** 紧凑型的最新公告。
    3.  **详细工作区：** 待办事项列表、看板、任务跟踪（占据较大面积）。

**2. 优化“空状态 (Empty State)”的展示逻辑**
*   **现状：** “看板”和“公告”即使在没有数据时，也占用了巨大的固定高度，中间放着一个巨大的插画。这是一种极大的空间浪费。
*   **建议：**
    *   **动态高度：** 使用 Vue 的 `v-if` / `v-else` 进行逻辑判断。如果有数据，展开大卡片；如果没有数据，**将高度压缩到原本的 1/3 甚至 1/4**，只保留一行提示字和“发布”按钮即可。
    *   或者采用**折叠面板**逻辑，空数据时默认折叠。

**3. 消除重复的模块命名**
*   **现状：** 中间有一张卡片叫“待办事项”（显示数字3），最下方又有一个列表区域叫“待办事项”。这会让用户产生认知混乱。
*   **建议：** 区分概览和列表。中间的统计卡片可以改名为**“任务概览”**或直接是数字，下方的详细列表保持“待办事项”不变。
