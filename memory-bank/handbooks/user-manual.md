## 1. 读者与前提

### 1.1 产品是什么

Project Filum 面向约 50–100 人规模组织，在统一工作台内完成**任务协同**、**汇报流转**、**人事档案**、**消息通知**与**制度知识**查阅。任务相关的讨论与附件留在任务详情中；消息中心用于通知、回执与回到业务来源，不替代任务评论。

### 1.2 角色

| 界面显示 | 含义 |
| --- | --- |
| 管理员 | 平台最高权限：部门树、全员账号、系统级配置与异常兜底 |
| HR | 人员与档案治理；**不能**维护部门树（无「部门管理」菜单） |
| 员工 | 默认最小权限：处理本人相关任务、汇报与消息 |

部门负责人、汇报线领导等**数据范围**由组织关系与任务/汇报指派决定，不一定单独显示为第四种全局角色。

### 1.3 登录与账号开通（邀请制）

- **不提供**公开自助注册或审批式注册入口。
- **新用户**：由管理员或 HR 在「人员管理」中 **邀请注册**，将邀请链接发给对方；对方打开链接后在登录页 **设置密码并完成激活**。
- **直接创建**：管理员/HR 可创建已启用账号并设置初始密码（适用于现场开通）。
- **空库首次部署**：打开 `/login` 时若系统中尚无管理员，会出现 **系统初始化** 向导（三步：邮箱 → 密码 → 姓名/员工编号），用于创建第一个管理员账号。

### 1.4 壳层导航

登录后左侧为导航，分两组：

**通用**

| 侧栏名称 | 路径 |
| --- | --- |
| 总览 | `/overview` |
| 任务中心 | `/task-center` |
| 知识库 | `/knowledge-base` |
| 汇报中心 | `/reports` |
| 设置 | `/settings`（默认进入 `/settings/profile`） |

**特殊**（按角色显示）

| 侧栏名称 | 路径 | 谁能看到 |
| --- | --- | --- |
| 人员管理 | `/people` | 管理员、HR |
| 部门管理 | `/departments` | 仅管理员 |

顶栏显示 **AI 命令**、**消息铃铛**（未读角标）、当前用户邮箱与角色标签、**退出登录**。点击铃铛打开右侧 **消息抽屉**（未读列表、跳转来源、查看全部）。全页消息中心仍可通过 `/messages` 或抽屉底部链接进入。移动端可通过菜单按钮展开侧栏。

## 2. 全局能力

### 2.1 AI 命令（顶栏）

**入口**: 顶栏 **「AI 命令」** 按钮。

**作用**: 打开对话框，输入以 `@系统` 或 `/` 开头的自然语言指令，由系统路由到知识检索等工具并返回说明性结果。适合快速查制度、SOP，**不能**替代在任务/汇报页面完成正式业务提交。

**步骤**:

1. 点击 **AI 命令**。
2. 在输入框填写问题（例如：`@系统 入职需要准备什么材料？`）。
3. 提交后查看返回内容与引用说明。

### 2.2 浏览器推送与 PWA（设置页）

在 **设置** 中可订阅浏览器推送、授权通知权限，并在支持的浏览器上 **安装到桌面**。详见 [第 9 章](#9-设置)。

### 2.3 旧书签与链接（自动跳转）

以下旧路径会跳转到新位置，书签可逐步更新：

| 旧路径 | 跳转到 |
| --- | --- |
| `/dashboard` | `/overview` |
| `/tasks` | `/task-center?filter=tracking` |
| `/task-templates` | `/task-templates`（侧栏独立入口；旧 `task-center?tab=templates` 会跳转至此） |
| `/approvals` | `/reports` |
| `/users`、`/profiles` | `/people`（带对应 tab 参数） |

## 3. 登录与账号激活

| 项目 | 说明 |
| --- | --- |
| **路径** | `/login` |
| **侧栏** | 无（未登录页） |

### 3.1 谁能使用

所有未登录用户；登录成功后进入总览。

### 3.2 页面上有什么

登录页按场景**自动切换**为以下三种之一（不再使用 Tab 切换）：

| 场景 | 触发条件 | 界面 |
| --- | --- | --- |
| **A 日常登录** | 无 `invite` 参数，且系统已初始化 | 邮箱 + 密码 + **登录**；底部灰色提示「本系统采用邀请制，请联系 HR 获取账号」 |
| **B 邀请激活** | URL 含 `?invite=<token>` | 全屏「欢迎加入，请设置密码」卡片，展示邀请邮箱/角色/有效期 |
| **C 系统初始化** | 空库（尚无管理员） | 全屏「系统初始化」三步向导 |

密码不符合安全策略（长度、大小写、数字、符号等）时，界面会显示**具体中文原因**，不再出现空白错误提示。

### 3.3 常用操作

#### 日常登录

1. 打开系统地址，进入登录页（场景 A）。
2. 填写 **邮箱**、**密码**。
3. 点击 **登录**。
4. **预期结果**: 提示登录成功，进入 **总览**；若登录前访问过其他页面，会回到该页面。

#### 首次初始化管理员（空库）

1. 确认后端与数据库已启动，且系统中尚无管理员。
2. 打开 `/login`，页面自动进入 **系统初始化** 向导（场景 C）。
3. 按步骤填写：管理员邮箱 → 密码与确认密码 → 姓名与员工编号。
4. 提交 **初始化管理员**。
5. **预期结果**: 提示初始化成功，以管理员身份进入总览。

#### 邀请链接激活

1. 打开 HR/管理员发来的邀请链接（地址中含 `invite=...`）。
2. 页面自动进入 **欢迎加入，请设置密码**（场景 B），展示预分配的邮箱、角色等信息。
3. 设置 **密码** 与 **确认密码**（须一致，且符合安全策略）。
4. 提交 **完成注册**。
5. **预期结果**: 提示注册成功并自动登录。

### 3.4 与其他页面的关系

- 登录后默认进入 **总览**；无权限访问的页面（如员工访问 `/departments`）会被重定向到总览。

### 3.5 常见疑问

- **看不到系统初始化向导？** 说明库中已有管理员，请使用日常登录或邀请链接。
- **邀请链接无效？** 可能已过期、已撤销或已完成注册；请联系管理员重新生成邀请。

## 4. 总览

| 项目 | 说明 |
| --- | --- |
| **路径** | `/overview` |
| **侧栏** | 总览 |

### 4.1 谁能看到

所有已登录用户。

### 4.2 页面上有什么

- **顶栏倒计时**（`DeadlineCountdown`，有未来截止任务时）：显示距最近截止任务的 `HH:MM:SS`，点击跳转任务中心 **跟踪** 视图。
- **消息预览**（`overview-widget-messages`）：最近若干条通知，点击打开顶栏 **铃铛** 消息 Drawer。
- **公告 / 白板**（`overview-widget-announcement-board`）：单卡片内 Toggle 切换；有权限时可 **发布公告** 或 **发布看板卡片**；管理员可 **归档**。
- **待办与汇报**（`overview-widget-todos`）：待办任务与待审汇报各展示 Top 5，点击跳转 **任务中心**（`filter=inbox` + `selected=`）或 **汇报中心**（`filter=pending` + `selected=`）。汇报数据加载失败时，待办任务区块仍会展示（总览其余区块独立加载）。
- **快捷入口**：任务中心、汇报中心、消息中心、知识库等；管理员/HR 另有 **人员管理** 链接。

### 4.3 常用操作

#### 从总览处理待办或汇报

1. 在 **待办与汇报** 小组件中点击某条任务或汇报。
2. **预期结果**: 打开 **任务中心**（待处理 + 选中任务）或 **汇报中心**（待处理 + 选中汇报）。

#### 从总览查看消息

1. 在 **消息预览** 中点击某条通知，或点击顶栏 **铃铛**。
2. **预期结果**: 右侧消息 Drawer 打开；可 **回到来源** 或 **查看全部消息**。

#### 发布看板卡片（有权限时）

1. 在 **公告 / 白板** 区域切换到 **白板**，点击 **发布看板卡片**。
2. 选择范围（如全公司或某部门）、填写标题与 Markdown 正文。
3. 提交。
4. **预期结果**: 提示看板卡片已发布，列表刷新。

#### 发布公告（有权限时）

1. 在 **公告 / 白板** 区域（默认 **公告**）点击 **发布公告**。
2. 选择发布部门、标题与正文。
3. 提交。
4. **预期结果**: 提示公告已发布。

#### 归档看板或公告（管理员）

1. 在对应卡片或公告上选择 **归档**（或撤回公告）。
2. **预期结果**: 该项从当前总览列表移除或标记为已归档。

### 4.4 与其他页面的关系

- 总览是登录后默认首页；数字来自任务中心聚合，不替代任务详情操作。
- 可通过 `?announcement=<id>` 高亮某条公告（深链）。

### 4.5 常见疑问

- **待办数字与任务中心不一致？** 刷新页面；若仍异常，可能是权限或任务状态刚变更，以任务中心列表为准。

## 5. 任务中心

| 项目 | 说明 |
| --- | --- |
| **路径** | `/task-center`（常用参数见下） |
| **侧栏** | 任务中心 |

本页采用 **Quick Chips 筛选 + Master-Detail 布局**：页头 **建立任务** 与 **列表/看板/甘特** 视图切换；左侧任务列表（约 35%）+ 右侧 **任务协同详情**（约 65%）。**个人备忘** 为全局右下角浮窗（任意页可用）；**任务模板** 为侧栏「特殊模块」独立入口（管理员/HR）。

### 5.1 谁能看到

所有已登录用户；**建立任务**、模板实例化、跨部门指派等依赖账号的 **发布任务** 能力（组织范围由后端权限计算）。**任务模板** 菜单对管理员/HR 可见；具备 `can_manage_templates` 的用户仍可通过任务中心页头 **任务模板** 进入。

### 5.2 页面上有什么

**页头**

- 任务中心说明文案。
- **列表 / 看板 / 甘特** 视图切换（URL 参数 `view=list|board|gantt`，默认列表）。
- **任务模板**（有模板权限时）：跳转 `/task-templates`。
- **建立任务**（有权限时）：打开页头 **建立任务**，在居中对话框填写标题、描述、执行人、部门、优先级、截止时间并提交；关闭时若有未保存内容会提示确认。

**筛选 Chips（Quick Filters）**

- **需要我处理的**（`filter=inbox`，默认）：inbox 待办表格。
- **我参与的/跟踪**（`filter=tracking`）：跟踪中任务，含催办等操作。
- **已完成/历史**（`filter=history`）：已完成任务列表。

**Master-Detail（列表视图，默认）**

- **左列**：当前筛选下的任务表格；点击行选中任务并同步 URL `selected=<任务ID>`。
- **右列**：**任务协同详情** 面板（选中任务后）：基本信息、握手状态、交付与验收、关注人、**任务资料附件**、评论、活动时间线、图任务时的 **工作流节点追踪**。

**看板 / 甘特视图**

- 切换页头 **看板** 或 **甘特** 后，全宽展示对应多视图（与列表共享同一任务数据源）；右侧仍为任务详情面板。

**全局个人备忘**

- 任意已登录页面右下角 **备忘** 浮动按钮 → 展开列表面板；**新建备忘** / 点击条目 **编辑** 打开独立对话框，可填可选标题、正文、关联任务、置顶；不再占用任务中心标签页。

**任务模板（独立页 `/task-templates`）**

- 图模板列表（名称、编码、类型、版本）与 **实例化** 派发；选模板 → 填写 launch 信息 → 确认派发后进入任务中心 **跟踪**。
- 入口：侧栏 **特殊模块 → 任务模板**（管理员/HR），或任务中心页头 **任务模板**（需 `can_manage_templates`）。
- Legacy 工作流 E 模板管理 UI 已移除（@ `0.89.0`）；后端 API 仍保留，见 `known-issues.md`。
- **规划中增强**（TCE）：列表与详情状态更一致、看板显示姓名、实例化默认所属部门等 — 见 [`plans/task-center-enhance.md`](../plans/task-center-enhance.md)。

### 5.3 常用操作

#### 建立任务并指派给他人

1. 点击页头 **建立任务**（居中对话框，非侧栏抽屉）。
2. 填写 **任务标题**（必填）、描述、**执行人**、所属部门、优先级、截止时间。
3. 点击提交。
4. **预期结果**: 对话框关闭，任务创建成功；执行人在 **待处理** 中可见（若为其当前处理人）。

#### 处理待办：接受任务并开始（图任务常见）

1. 打开 **需要我处理的**，找到目标任务（或从消息中心 **回到来源**）。
2. 点击任务行，右侧出现详情（inbox 项也可直接打开详情，无需先切跟踪）。
3. 若显示 **接受任务**，点击并接受。
4. 若状态为待办且已接受，点击 **开始处理**（或界面上的状态推进按钮，文案可能为「开始处理」等等价操作）。
5. **预期结果**: 状态变为进行中；可进行后续交付。

#### 提交交付物（执行人）

1. 在任务详情中选择处于 **进行中** 且需要你交付的任务。
2. 在 **交付与验收** 区域填写 **交付说明**。
3. 点击 **提交交付物**。
4. **预期结果**: 任务进入待验收状态；创建人/验收人可在跟踪与详情中看到交付说明与时间。

#### 验收通过或打回返工（创建人或管理者）

1. 打开处于 **评审中** 的任务详情。
2. **验收通过**: 可选填写验收评价与 **完成质量评分**，点击 **验收通过**。
3. **打回返工**: 点击 **打回返工**，必填返工原因。
4. **预期结果**: 通过则任务完成；打回则执行人侧显示返工中并需重新提交交付。

#### 握手：退回协商、转办（图任务、待办态）

- **退回协商**: 点击 **退回协商**，填写协商原因 → 任务回到创建人侧调整。
- **转办**: 点击 **转办**，选择新执行人并填写原因 → 新执行人待确认。

（按钮仅在当前角色与任务状态满足时出现。）

#### 上传任务资料附件

1. 在任务详情 **任务资料附件** 区域点击 **选择附件**，选择符合要求的文件（见 [附录：附件类型](#122-附件类型与大小)）。
2. 点击 **上传到任务**。
3. **预期结果**: 列表中出现附件，可 **查看** / **下载**（经鉴权 API 拉取，视文件类型）。

#### 添加评论

1. 在 **评论与留痕** 填写评论；管理员可勾选 **内部备注**。
2. 可选添加 **评论附件**。
3. 点击 **提交评论**。
4. **预期结果**: 评论出现在活动时间线。

#### 催办（跟踪筛选）

1. 切换到 **我参与的/跟踪**，在表格对应行点击 **催办**。
2. **预期结果**: 系统向该任务写入催办类评论/通知（执行人可在消息或任务中看到）。

#### 个人备忘

1. 在任意页面点击右下角 **备忘** 浮动按钮。
2. 填写内容，可选关联任务、置顶，点击 **创建备忘** 或保存编辑。
3. **预期结果**: 备忘出现在面板列表；仅本人可见。

#### 使用任务模板（有权限时）

1. 侧栏打开 **任务模板**（或任务中心页头 **任务模板**），在列表中选择模板。
2. 点击 **实例化**，填写 launch 表单（如征集主题、负责人等），确认派发。
3. **预期结果**: 创建批次/制作 Run，在 **任务跟踪** 中可见 ROOT 与子任务。

#### 深链打开指定任务

- 地址形式：`/task-center?filter=tracking&selected=<任务ID>`（旧 `?tab=tracking` 仍兼容并自动跳转）
- **预期结果**: 进入对应筛选并选中任务详情（列表加载完成后生效）。

### 5.4 与其他页面的关系

- 任务讨论与附件在 **本页详情**，不在消息中心聊天。
- 消息通知可 **回到来源** 跳回任务中心。
- 总览卡片数字与 **待处理/跟踪** 列表同源。

### 5.5 常见疑问

- **待处理里有任务，但详情面板空白？** 点击左侧列表行选中任务；确认 URL 含 `selected=`。
- **待处理 vs 跟踪 vs 历史？** 需要我处理的 = 当前要你办；跟踪 = 与你相关且未结束；历史 = 已完成归档。
- **建立任务为什么不在列表区？** 已统一收到页头 **建立任务**。
- **备忘去哪了？** 已改为全局右下角浮窗，任意页面可用。
- **模板为什么不在任务中心里？** 已拆为侧栏 **任务模板** 独立页，避免与日常执行混排。

## 7. 汇报中心

| 项目 | 说明 |

| **侧栏** | 汇报中心 |

本页采用 **邮件客户端式 Master-Detail**：左侧汇报列表 + 右侧阅读区；**撰写汇报** 为单一 Drawer，收件人下拉隐式决定向上/向下方向。

### 7.1 谁能看到

所有已登录用户；**撰写汇报** 是否可用取决于是否存在可用的向上/向下目标链路（界面会提示「暂无可发起链路」）。

### 7.2 页面上有什么

- 页头：待处理 / 我发起 / 历史归档数量摘要。
- **撰写汇报** 按钮（统一入口，打开右侧 Drawer）。
- **筛选 Chips**：待处理、我发起、历史归档。
- **左侧列表**：标题、对方姓名、状态、时间；点击选中并同步 URL `selected=<汇报ID>`。
- **右侧阅读区**：正文、附件、路由时间线、**可用操作按钮**（由系统按角色与状态生成）。

**撰写汇报 Drawer（一步完成）**

- **收件人**（下拉）：选项形如 `上级 - 张三 (向上)` / `下级 - 研发组 (向下)`，系统自动推断方向。
- **主题**、**正文**、可选 **挂接审批流**、**附件**。

### 7.3 常用操作

#### 撰写并发送汇报

1. 点击 **撰写汇报**。
2. 在 **收件人** 下拉中选择对象（向上或向下由选项隐式决定）。
3. 填写标题、正文；可选选择审批流定义、添加附件。
4. 点击 **发送汇报**。
5. **预期结果**: 提示已发起；在 **我发起** 中可见，接收方在 **待处理** 中可见。

#### 处理待办汇报

1. 确认 **待处理** 筛选已选中，在左侧列表点击目标汇报。
2. 在右侧阅读区阅读正文与时间线。
3. 点击 **操作按钮**（文案以界面为准，如继续上报、确认完成、退回等）。
4. **预期结果**: 汇报沿汇报线流转或结束；状态与时间线更新。

#### 查看历史

1. 点击 **历史归档** 筛选，在左侧选择汇报。
2. **预期结果**: 右侧显示已完成、已归档或已退回的汇报详情与可用操作（如归档）。

### 7.4 与其他页面的关系

- 相关通知在 **消息中心**，可通过 **回到来源** 返回汇报页（深链可带 `?filter=&selected=`）。
- 旧路径 `/approvals` 自动进入本页。

### 7.5 常见疑问

- **为什么有时不能撰写汇报？** 当前账号没有配置好的向上/向下目标（组织或汇报线未维护）。
- **还要先选向上还是向下吗？** 不需要；选 **收件人** 即可，方向由系统自动判断。
- **为什么操作按钮在右侧？** Master-Detail 布局：左侧选汇报，右侧集中阅读与操作，避免列表与详情来回跳转。
