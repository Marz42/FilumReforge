## 1. 读者与前提

### 1.1 产品是什么

Project Filum 面向约 50–100 人规模组织，在统一工作台内完成**任务协同**、**汇报流转**、**人事档案**、**消息通知**与**制度知识**查阅。任务相关的讨论与附件留在任务详情中；消息中心用于通知、回执与回到业务来源，不替代任务评论。

### 1.2 角色

| 界面显示 | 含义 |
| --- | --- |
| 管理员 | 平台最高权限：部门树、全员账号、系统级配置与异常兜底 |
| HR | 人员与档案治理；**不能**维护部门树（无「部门管理」菜单） |
| 员工 | 默认最小权限：处理本人相关任务、汇报与消息 |

部门负责人、汇报线领导等**数据范围**由组织关系与任务/汇报指派决定，不一定单独显示为第四种全局角色。

### 1.3 登录与账号开通（邀请制）

- **不提供**公开自助注册或审批式注册入口。
- **新用户**：由管理员或 HR 在「人员管理」中 **邀请注册**，将邀请链接发给对方；对方打开链接后在登录页 **设置密码并完成激活**。
- **直接创建**：管理员/HR 可创建已启用账号并设置初始密码（适用于现场开通）。
- **空库首次部署**：打开 `/login` 时若系统中尚无管理员，会出现 **系统初始化** 向导（三步：邮箱 → 密码 → 姓名/员工编号），用于创建第一个管理员账号。

### 1.4 壳层导航

登录后左侧为导航，分两组：

**通用**

| 侧栏名称 | 路径 |
| --- | --- |
| 总览 | `/overview` |
| 任务中心 | `/task-center` |
| 知识库 | `/knowledge-base` |
| 汇报中心 | `/reports` |
| 设置 | `/settings`（默认进入 `/settings/profile`） |

**特殊**（按角色显示）

| 侧栏名称 | 路径 | 谁能看到 |
| --- | --- | --- |
| 人员管理 | `/people` | 管理员、HR |
| 部门管理 | `/departments` | 仅管理员 |

顶栏显示 **AI 命令**、**消息铃铛**（未读角标）、当前用户邮箱与角色标签、**退出登录**。点击铃铛打开右侧 **消息抽屉**（未读列表、跳转来源、查看全部）。全页消息中心仍可通过 `/messages` 或抽屉底部链接进入。移动端可通过菜单按钮展开侧栏。

## 2. 全局能力

### 2.1 AI 命令（顶栏）

**入口**: 顶栏 **「AI 命令」** 按钮。

**作用**: 打开对话框，输入以 `@系统` 或 `/` 开头的自然语言指令，由系统路由到知识检索等工具并返回说明性结果。适合快速查制度、SOP，**不能**替代在任务/汇报页面完成正式业务提交。

**步骤**:

1. 点击 **AI 命令**。
2. 在输入框填写问题（例如：`@系统 入职需要准备什么材料？`）。
3. 提交后查看返回内容与引用说明。

### 2.2 浏览器推送与 PWA（设置页）

在 **设置** 中可订阅浏览器推送、授权通知权限，并在支持的浏览器上 **安装到桌面**。详见 [第 9 章](#9-设置)。

### 2.3 旧书签与链接（自动跳转）

以下旧路径会跳转到新位置，书签可逐步更新：

| 旧路径 | 跳转到 |
| --- | --- |
| `/dashboard` | `/overview` |
| `/tasks` | `/task-center?tab=tracking` |
| `/task-templates` | `/task-center?tab=templates` |
| `/approvals` | `/reports` |
| `/users`、`/profiles` | `/people`（带对应 tab 参数） |

## 3. 登录与账号激活

| 项目 | 说明 |
| --- | --- |
| **路径** | `/login` |
| **侧栏** | 无（未登录页） |

### 3.1 谁能使用

所有未登录用户；登录成功后进入总览。

### 3.2 页面上有什么

登录页按场景**自动切换**为以下三种之一（不再使用 Tab 切换）：

| 场景 | 触发条件 | 界面 |
| --- | --- | --- |
| **A 日常登录** | 无 `invite` 参数，且系统已初始化 | 邮箱 + 密码 + **登录**；底部灰色提示「本系统采用邀请制，请联系 HR 获取账号」 |
| **B 邀请激活** | URL 含 `?invite=<token>` | 全屏「欢迎加入，请设置密码」卡片，展示邀请邮箱/角色/有效期 |
| **C 系统初始化** | 空库（尚无管理员） | 全屏「系统初始化」三步向导 |

密码不符合安全策略（长度、大小写、数字、符号等）时，界面会显示**具体中文原因**，不再出现空白错误提示。

### 3.3 常用操作

#### 日常登录

1. 打开系统地址，进入登录页（场景 A）。
2. 填写 **邮箱**、**密码**。
3. 点击 **登录**。
4. **预期结果**: 提示登录成功，进入 **总览**；若登录前访问过其他页面，会回到该页面。

#### 首次初始化管理员（空库）

1. 确认后端与数据库已启动，且系统中尚无管理员。
2. 打开 `/login`，页面自动进入 **系统初始化** 向导（场景 C）。
3. 按步骤填写：管理员邮箱 → 密码与确认密码 → 姓名与员工编号。
4. 提交 **初始化管理员**。
5. **预期结果**: 提示初始化成功，以管理员身份进入总览。

#### 邀请链接激活

1. 打开 HR/管理员发来的邀请链接（地址中含 `invite=...`）。
2. 页面自动进入 **欢迎加入，请设置密码**（场景 B），展示预分配的邮箱、角色等信息。
3. 设置 **密码** 与 **确认密码**（须一致，且符合安全策略）。
4. 提交 **完成注册**。
5. **预期结果**: 提示注册成功并自动登录。

### 3.4 与其他页面的关系

- 登录后默认进入 **总览**；无权限访问的页面（如员工访问 `/departments`）会被重定向到总览。

### 3.5 常见疑问

- **看不到系统初始化向导？** 说明库中已有管理员，请使用日常登录或邀请链接。
- **邀请链接无效？** 可能已过期、已撤销或已完成注册；请联系管理员重新生成邀请。

### 审阅批注区

- 需要细化一下前端的提示，例如现在注册或者更改密码时，如果不符合要求（大小写混合+数字）弹出的错误提示是空的，需要进一步细化各个错误提示，给出完整的错误原因，方便用户使用；
- 场景 A（正常）：只显示“登录”表单。底部加一行灰色小字：“本系统采用邀请制，请联系 HR 获取账号”（解决你批注中的疑问）。
- 场景 B（带 invite 参数）：直接全屏显示“欢迎加入，请设置密码”的激活卡片，隐藏普通登录入口。
- 场景 C（后端检测到空库）：全屏显示“系统初始化”向导页面。

## 4. 总览

| 项目 | 说明 |
| --- | --- |
| **路径** | `/overview` |
| **侧栏** | 总览 |

### 4.1 谁能看到

所有已登录用户。

### 4.2 页面上有什么

- **我的待办** / **跟踪任务** 数量卡片（点击可跳到任务中心对应标签）。
- **快捷入口**：任务中心、消息中心、知识库等；管理员/HR 另有 **人员管理** 入口。
- **看板卡片**：按部门范围展示的信息卡片（有发布权限者可新建）。
- **公告**：组织公告列表（有发布权限者可发布；管理员可归档）。

### 4.3 常用操作

#### 从总览进入待办或跟踪

1. 在 **我的待办** 或 **跟踪任务** 卡片上点击 **查看待办** / **查看跟踪**。
2. **预期结果**: 打开 **任务中心**，并切换到 **待处理** 或 **任务跟踪** 标签。

#### 发布看板卡片（有权限时）

1. 在看板区域点击 **发布看板卡片**（或等价入口）。
2. 选择范围（如全公司或某部门）、填写标题与 Markdown 正文。
3. 提交。
4. **预期结果**: 提示看板卡片已发布，列表刷新。

#### 发布公告（有权限时）

1. 在公告区域点击 **发布公告**。
2. 选择发布部门、标题与正文。
3. 提交。
4. **预期结果**: 提示公告已发布。

#### 归档看板或公告（管理员）

1. 在对应卡片或公告上选择 **归档**（或撤回公告）。
2. **预期结果**: 该项从当前总览列表移除或标记为已归档。

### 4.4 与其他页面的关系

- 总览是登录后默认首页；数字来自任务中心聚合，不替代任务详情操作。
- 可通过 `?announcement=<id>` 高亮某条公告（深链）。

### 4.5 常见疑问

- **待办数字与任务中心不一致？** 刷新页面；若仍异常，可能是权限或任务状态刚变更，以任务中心列表为准。

## 5. 任务中心

| 项目 | 说明 |
| --- | --- |
| **路径** | `/task-center`（常用参数见下） |
| **侧栏** | 任务中心 |

本页是 IA 审阅重点：同一侧栏入口下包含 **四个标签**、页头 **建立任务**、以及嵌入的 **任务协同详情**（列表/看板/甘特 + 右侧面板）与 **任务模板** 工作台。

### 5.1 谁能看到

所有已登录用户；**建立任务**、模板管理、跨部门指派等依赖账号的 **发布任务** 能力（组织范围由后端权限计算）。

### 5.2 页面上有什么

**页头**

- 任务中心说明文案。
- **建立任务**（有权限时）：打开右侧抽屉，填写标题、描述、执行人、部门、优先级、截止时间并提交。

**标签一：待处理**

- 表格：任务标题、优先级、状态、部门、**当前阶段**、**当前处理人**、截止时间。
- 仅展示需要你处理或待你确认的 inbox 项。

**标签二：任务跟踪**

- 上方表格：与你相关的跟踪中任务（含关联方式、交付信号、**催办** 按钮等）。
- 下方 **任务协同详情** 区域（`TasksView`）：
  - 左：任务列表 / **看板** / **甘特** 切换；统计卡片（任务总数、完成率等）。
  - 右：**任务协同详情** 面板（选中任务后）：基本信息、握手状态、交付与验收、关注人、**任务资料附件**、评论、活动时间线、图任务时的 **工作流节点追踪**。
- 再下方：**历史任务** 表格（已完成等）。

**标签三：备忘**

- 左侧：新增/编辑个人备忘（内容、可选关联任务、置顶）。
- 右侧：置顶备忘与其他备忘列表。

**标签四：任务模板**

- 模板列表与 **模板详情**（步骤设计、启用/停用、编辑、删除、**实例化**、调度等）；面向具备模板管理权限的用户。

### 5.3 常用操作

#### 建立任务并指派给他人

1. 点击页头 **建立任务**。
2. 填写 **任务标题**（必填）、描述、**执行人**、所属部门、优先级、截止时间。
3. 点击提交。
4. **预期结果**: 抽屉关闭，任务创建成功；执行人在 **待处理** 中可见（若为其当前处理人）。

#### 处理待办：接受任务并开始（图任务常见）

1. 打开 **待处理**，找到目标任务（或从消息中心 **回到来源**）。
2. 切换到 **任务跟踪** 标签，在列表中点击该任务，右侧出现详情。
3. 若显示 **接受任务**，点击并接受。
4. 若状态为待办且已接受，点击 **开始处理**（或界面上的状态推进按钮，文案可能为「开始处理」等等价操作）。
5. **预期结果**: 状态变为进行中；可进行后续交付。

#### 提交交付物（执行人）

1. 在任务详情中选择处于 **进行中** 且需要你交付的任务。
2. 在 **交付与验收** 区域填写 **交付说明**。
3. 点击 **提交交付物**。
4. **预期结果**: 任务进入待验收状态；创建人/验收人可在跟踪与详情中看到交付说明与时间。

#### 验收通过或打回返工（创建人或管理者）

1. 打开处于 **评审中** 的任务详情。
2. **验收通过**: 可选填写验收评价与 **完成质量评分**，点击 **验收通过**。
3. **打回返工**: 点击 **打回返工**，必填返工原因。
4. **预期结果**: 通过则任务完成；打回则执行人侧显示返工中并需重新提交交付。

#### 握手：退回协商、转办（图任务、待办态）

- **退回协商**: 点击 **退回协商**，填写协商原因 → 任务回到创建人侧调整。
- **转办**: 点击 **转办**，选择新执行人并填写原因 → 新执行人待确认。

（按钮仅在当前角色与任务状态满足时出现。）

#### 上传任务资料附件

1. 在任务详情 **任务资料附件** 区域点击 **选择附件**，选择符合要求的文件（见 [附录：附件类型](#122-附件类型与大小)）。
2. 点击 **上传到任务**。
3. **预期结果**: 列表中出现附件，可 **查看** / **下载**（视文件类型）。

#### 添加评论

1. 在 **评论与留痕** 填写评论；管理员可勾选 **内部备注**。
2. 可选添加 **评论附件**。
3. 点击 **提交评论**。
4. **预期结果**: 评论出现在活动时间线。

#### 催办（跟踪表）

1. 在 **任务跟踪** 表格对应行点击 **催办**。
2. **预期结果**: 系统向该任务写入催办类评论/通知（执行人可在消息或任务中看到）。

#### 个人备忘

1. 打开 **备忘** 标签，填写内容，可选关联任务、置顶。
2. 点击 **创建备忘** 或保存编辑。
3. **预期结果**: 备忘出现在右侧列表；仅本人可见。

#### 使用任务模板（有权限时）

1. 打开 **任务模板** 标签，选择模板查看步骤与流转关系。
2. **新建模板** / **编辑模板** / **启用·停用** / **实例化** 等按界面按钮操作。
3. **预期结果**: 实例化后产生一批按模板推进的任务，在跟踪与待办中可见。

#### 深链打开指定任务

- 地址形式：`/task-center?tab=tracking&selected=<任务ID>`
- **预期结果**: 进入任务跟踪并选中对应任务详情（列表加载完成后生效）。

### 5.4 与其他页面的关系

- 任务讨论与附件在 **本页详情**，不在消息中心聊天。
- 消息通知可 **回到来源** 跳回任务中心。
- 总览卡片数字与 **待处理/跟踪** 列表同源。

### 5.5 常见疑问

- **待处理里有任务，但跟踪里点不到详情？** 先在跟踪列表点击行；若任务仅出现在待办，处理完或切换标签后再看跟踪/历史。
- **待处理 vs 任务跟踪 vs 历史任务？** 待办 = 当前要你办；跟踪 = 与你相关且未结束的主列表 + 详情；历史 = 已结束归档视图。
- **建立任务为什么不在跟踪区？** 已统一收到页头 **建立任务**，避免两个入口。
- **模板标签和日常执行任务在一起好吗？** 当前设计如此，IA 审阅可决定是否拆为独立菜单。

### 审阅批注区

重构建议：彻底拆解复杂的 Tab，采用“过滤器 + 视图切换”模式。
当前把“日常待办”、“项目跟踪”、“个人备忘”、“后台模板”揉在一个页面的 4 个 Tab 里，逻辑极度割裂。
拆分非核心功能：“任务模板”是管理/配置行为，必须抽离，可放在右上角齿轮图标或单独抽成独立菜单。“备忘”属于个人功能，建议做成全局的侧边栏浮窗（类似便签），而不应占据主业务视窗。
合并与过滤（取消待处理/跟踪/历史的物理 Tab）：
采用现代协同软件的 Master-Detail（左侧列表-右侧详情） 布局。
顶部使用 快捷胶囊过滤器 (Quick Chips)：[需要我处理的 (Inbox)] [我参与的/跟踪的] [已完成/历史]。
视图平级切换：列表 (List) / 看板 (Board) / 甘特图 (Gantt) 应该是同一份数据的不同展示维度。将这三个切换按钮放在页面右上角，让用户自己决定用什么视角看任务。

**发起汇报弹窗（两步）**

1. **选择类型**：发起向上汇报 / 发起向下传达。
2. **填写表单**：目标对象、标题、正文、可选挂接审批流、**选择附件**（草稿附件，提交时一并绑定）。

1. 点击 **发起汇报** → **发起向下传达**。
2. 选择目标（下级或传达对象）、填写内容与附件。
3. 提交。
4. **预期结果**: 提示传达已发起；接收方在待处理中处理。

1. 打开 **待处理** 标签，阅读汇报正文与路由时间线。
2. 点击卡片上显示的 **操作按钮**（文案以界面为准，如继续上报、确认完成、退回等）。
3. **预期结果**: 汇报沿汇报线流转或结束；状态与时间线更新。

#### 查看历史

1. 打开 **历史归档** 标签。

#### 回到来源

1. 在抽屉跳转或全页详情点击 **回到来源**。
2. **预期结果**: 跳转到任务中心、汇报中心等来源页面（若消息含来源信息）；若无来源则提示不支持。

### 8.4 与其他页面的关系

- 浏览器推送在 **设置 → 通知偏好** 中订阅，与本页列表互补。

| 术语 | 含义 |
| --- | --- |
| 待处理（任务） | 任务中心 inbox：当前需要你处理或确认的任务 |

### 12.3 附件类型与大小

上传前界面会预检；服务端另做校验。允许类型包括：

- 图片：png、jpg、gif、webp 等  
- 文档：pdf、xlsx、txt、md、docx  
- 音频：mp3、wav  

大小上限（单文件）：文本类（含 txt/md/docx）**10MB**；音频 **50MB**；其余允许类型 **25MB**。
